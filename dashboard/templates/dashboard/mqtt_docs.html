📡 Inbound Telemetry

Dynamic Topic: {{ app_settings.mqtt_topic_helmet_status|default:"helmet/+/status" }}

The helmet should publish a JSON payload every 5-10 seconds to the status topic (replace + with Helmet ID).

{
  "speed": 42.5,
  "lat": 36.7525,
  "lon": 3.04197,
  "alc": 0.02,
  "worn": true,
  "strapped": true,
  "tilt": -5.2,
  "bat": 88,
  "rssi": 92,
  "latency": 24
}
QoS 1 Retained: No

⚙️ Configuration Request

Dynamic Topic: helmet/+/request

ESP can request current system limits and configurations.

Request Body:

{ "type": "get_settings" }

Response: (on helmet/+/settings)

{
  "allowed_alcohol_level": {{ app_settings.allowed_alcohol_level }},
  "speed_limit": {{ app_settings.speed_limit|default:60 }},
  "refresh_rate": {{ app_settings.map_refresh_rate_seconds }}
}

🌐 Web Integration & Remote Hosting

Cloudflare & WebSocket Secure (WSS) Configuration

WebSocket Connectivity

To view live telemetry on the dashboard over HTTPS, you must use a WebSocket Secure (WSS) bridge.

  • Host: {{ app_settings.mqtt_websocket_host|default:"[Set in Settings]" }}
  • Port: {{ app_settings.mqtt_websocket_port|default:443 }}
  • Protocol: {% if app_settings.mqtt_use_ssl %}wss://{% else %}ws://{% endif %}

Cloudflare Setup

When using Cloudflare, ensure your MQTT broker is accessible via a subdomain with WebSockets enabled in the Cloudflare dashboard.

Tip: Use port 443 for WSS traffic through Cloudflare proxies to ensure maximum compatibility with modern browsers.

Self-Healing Architecture

The backend listener implements an exponential backoff retry mechanism. If the broker is unreachable, the system will attempt to reconnect every 5 seconds without crashing the main Django service.
